Preguntas Lab 1:

  • ¿Cuál es la diferencia entre Internet y la World Wide Web?
    • El internet es la red que de cierta forma potencia todos los servicios en linea como llamadas, partidas en linea, streaming, etc. Mientras que la Web son los protocolos usa el Internet para vincular sitios.
  • ¿Cuáles son las partes de un URL?
    • Protocolo: Formato que intercambia datos entre los dispositivos involucrados (http, ftp, etc.)
    • Servidor: La maquina de destino (Google, YouTube, Nintendo, etc.), el dominio (.com, .gov, .mx), y el subdominio (www)
    • Puerto: Número que corresponde a la entrada de la máquina.
    • Directorio: Igual que un directorio de un sistema operativo, organiza las paginas del servidor (servidor/inicio/preguntas/)
    • Página: La pagina actual
  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET: Solicita un recurso, solo recupera datos.
    • HEAD: Igual a GET, solo que este no obtiene el cuerpo de la respuesta.
    • POST: Envia una entidad a un recurso, usualmente haciendo cambios al servidor.
    • PUT: Reemplaza las representaciones actuales del recurso de destino con la carga útil.
    • PATCH: Aplica modificaciones a un recurso.
    • DELETE: Borra un recurso.
  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
      Se usa POST debido a que envia nuestra respuesta, la cual es verificada en el servidor, lo que permitira el acceso a este mismo, esto ultimo siendo el cambio al servidor.
  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
      Se usa GET debido a que obtiene la pagina.
  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?
      El codigo 200 representa un OK, es decir, que una solicitud se realizó con éxito, sin errores.
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?
      Si, debido a que el codigo de error 404 ocurre cuando no se puede encontrar el contenido que se solicita.
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?
      Debido a que según el sitio de desarroladores de Mozilla, el código 500 indica que no encontró un mejor código de los 5XX y los administradores solicitan más detalles para conocer el origen del error, pero en general sí.
  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.
      Significa que ya no es recomendado su uso en HTML5 debido a una actualización del estandar. Algunos elementos desaprobados de HTML4 son basefont, font, center, big en favor de las hojas de estilo css.
  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?
      HTML5 puede definir un juego de caracteres, permite usar elementos SVG y MathML, introduce header, footer, section, etc., y desaprueba basefont, big, center, etc.
  • ¿Qué componentes de estructura y estilo tiene una tabla?
      Una tabla de html tiene elementos de row, celda, y un encabezado común, también se puede cambiar el estilo de las celdas, es decir, sus bordes, color de relleno, etc.
  • ¿Cuáles son los principales controles de una forma HTML5?
      Existen los botones, las listas de selección multiple, el input de texto, sliders, etc.

  • ¿Cuál es el ciclo de vida de los sistemas de información?
    • Fase de Planeación: Desde montar el sistema hasta definir el cronograma y la asignación de tareas.
    • Fase de Análisis: Se analizan los requisitos de lo que debe hacer el sistema, en esta fase se identifica lo esencial para evitar usar recursos en aquello que no sea de esencia.
    • Fase de Diseño: Se verifica cómo el sistema cumplirá con las necesidades que se tengan.
    • Fase de Implementación: Se empieza a armar el sistema, por lo que se recomienda tener al equipo con un mismo enfoque.
    • Fase de Pruebas: Se realizan… pruebas para asegurar que el sistema cumpla con los requisitos establecidos, de preferencia a satisfacción del cliente.
    • Fase de Despliegue: Se implementan el hardware y software necesarios para poner en marcha el nuevo sistema con el fin de asegurar que no haya problemas y documentarlo para sus futuros usuarios.
    • Fase de Mantenimiento: A medida que pasa el tiempo se le va dando soporte al sistema para mejorar su rendimiento y arreglar fallas que vayan apareciendo.
    • Fase de Cierre: Dependiendo de las circunstancias se migren los datos a un nuevo sistema de información para su uso en un nuevo sistema y dejar de darle soporte, o simplemente iniciar el proceso de cierre.
  • ¿Cuál es el ciclo de desarrollo de sistemas de información?
    • Fase de Planeación: Se identifica la necesidad de un nuevo software para lograr los objetivos de la empresa, averiguando el alcance del problema y buscar sus soluciones.
    • Fase de Requerimientos: Se trabaja en la fuente del problema o necesidad, se consideran los requisitos funcionales del proyecto, se hacen análisis del sistema y de las necesidades de los usuarios finales.
    • Fase de Diseño: Se describen a detalle las especificaciones, caracterisiticas, y operaciones que satisfagan los requisitos funcionales.
    • Fase de Desarrollo: Se empieza a construir el sistema, esto hace referenica al código y los diagramas de flujo, para asegurar que el proceso sea organizdo correctamente.
    • Fase de Integración: Se realizan… pruebas de QA para asegurar que el sistema cumpla con los requisitos establecidos, y que el software en si sea de calidad. Se repite hasta que el cliente lo considere aceptable.
    • Fase de Implementación: Contrario a lo que el nombre de a pensar, aquí es donde se hace la mayor parte del código, también se hace la instalación real del sistema desarrollado, si es un sistema que reemplazara uno viejo, también se hace la migración en esta fase.
    • Fase de Mantenimiento: Se realiza mantenimiento y se realizan actualizaciones periodicamente. Los usuarios finales ajustan el sistema para mejorar su rendimiento, y pueden agregar nuevos requisitos que vayan apareciendo.

Preguntas Lab 3:

  • Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?
    • Mi recomendación es que se use con cuidado y cuando se quiere dar prioridad a algo, normalmente la prioridad que se da en cuanto al estilo denominado por un CSS va en orden de ID, Clase, Elemento, y el selector universal (*) al final, sin embargo, usar !important hace que el atributo marcado como importante se tome como la más alta prioridad sin importar donde se definió.
  • Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?
    • Se debe tomar en cuenta el tamaño de la imagen (en terminos informáticos), ya que mientras más grande sea su tamaño, más tardará en cargarse.
  • Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    • Hay que usar px, ya que px siempre hace referencia al tamaño de la pantalla, mientras que pt y % siempre serán relativas a otra unidad.
  • ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?
    • Debido a que se eliminan cosas que el sistema no necesita realmente, como las indentaciones, espacios en blanco, etc. A nosotros nos sirve para tener las cosas organizadas, pero la computadora recorre el código caracter por caracter, por lo que reducir la cantidad mejora el rendimiento.
